# sp-webdav-tool

English version: [README.md](README.md)

Используйте [Super Productivity](https://super-productivity.com) как собственную серверную систему задач для CLI, автоматизаций и AI-агентов.

Проект предоставляет небольшую Python-библиотеку и CLI для чтения и изменения данных Super Productivity через уже настроенную WebDAV-синхронизацию — без браузерной сессии, без плагинов и без стороннего облака.

## Зачем нужен этот проект

Super Productivity построен как local-first приложение: данные остаются под вашим контролем, а синхронизацию можно организовать через собственное WebDAV-хранилище. Этот проект делает те же самые данные задач доступными для скриптов, cron-задач и серверных агентов, таких как Hermes. 

## Почему не MCP?

Этот проект решает другую задачу.

Интеграции через MCP удобны, когда AI-ассистент работает с уже запущенным приложением через специальный мост. Этот инструмент предназначен для серверной автоматизации и headless-сценариев: он работает напрямую с уже существующим WebDAV-хранилищем синхронизации, без браузерной сессии, без десктопного интерфейса и без plugin bridge.

Выбирайте этот проект, если вам нужны:
- CLI-скрипты и cron-автоматизация
- серверные агенты, например Hermes
- собственная схема работы без зависимости от браузера
- прямой доступ к тому же sync storage, которое уже используют ваши устройства

## Как это работает

Super Productivity синхронизирует данные задач через файловую модель хранения. Этот инструмент читает и изменяет те же данные синхронизации в WebDAV, которые уже используют ваши десктопные и мобильные клиенты.

На практике инструмент:
- скачивает текущий файл синхронизации,
- вносит только запрошенное изменение,
- проверяет, не изменил ли этот файл другой клиент за время работы,
- при конфликте не перезаписывает более новые данные молча, а повторяет попытку,
- создаёт резервные копии перед операциями записи.

## Важное замечание

Сейчас Super Productivity использует файловую синхронизацию. Этот инструмент изменяет те же данные, что и десктопные и мобильные клиенты, поэтому при одновременной записи с нескольких устройств конфликты всё равно возможны.

Чтобы снизить этот риск, библиотека проверяет версию файла перед записью и создаёт автоматические резервные копии. Это делает работу безопаснее, но не заменяет полноценный движок бесконфликтной синхронизации. 

## Требования

- Python 3.11+
- рекомендуется [uv](https://docs.astral.sh/uv/)
- Super Productivity с включённой WebDAV-синхронизацией

## Установка

```bash
uv sync
```

## Настройка

Создайте `.env` на основе `.env.example`:

```bash
cp .env.example .env
```

Пример:

```env
SP_WEBDAV_URL=https://sp.example.com:51434
SP_WEBDAV_PATH=/webdav/MAIN.json
SP_WEBDAV_USERNAME=your_username
SP_WEBDAV_PASSWORD=your_password
```

## CLI

```bash
uv run sp-webdav tasks list
uv run sp-webdav tasks add "Проверить деплой"
uv run sp-webdav tasks done <префикс-id>
uv run sp-webdav projects list
uv run sp-webdav backup
```

## Разработка

```bash
uv sync --extra dev
uv run ruff check .
uv run mypy src tests
uv run pytest -v
```
